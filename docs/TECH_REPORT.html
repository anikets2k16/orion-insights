ORION DeepSearch

Technology, LLM, and Architecture Overview

LangGraph FastAPI TanStack Start Supabase Deterministic AI

1. Executive Summary

ORION DeepSearch is a deterministic, multi-agent research platform. A LangGraph orchestrator coordinates 9 specialized agents through an 8-step pipeline, backed by FastAPI + Celery on the server and a TanStack Start v1 React 19 app on the client. Its headline differentiator is byte-identical reproducibility of every research run.

4. Agent Pipeline

A LangGraph state machine drives 9 agents through 8 sequential steps, with branching for the 3 persona crews (Researcher, Analyst, Reporter).

Client → FastAPI → LangGraph
        ├─ PlannerAgent
        ├─ RetrievalAgent      ─┐
        ├─ WebSearchAgent       │  parallel
        ├─ AcademicAgent       ─┘
        ├─ SynthesisAgent
        ├─ CritiqueAgent
        ├─ GuardrailAgent
        ├─ ReportAgent
        └─ MemoryAgent
      → Celery worker → Supabase + Redis

5. Determinism (NFR-1)

Every research run is reproducible byte-for-byte given the same input. This is the project's headline non-functional requirement.
  • SHA-256 content-addressed completion cache
  • temperature=0, fixed seed on every call
  • Snapshot-only model enforcement (CI fails on floating aliases)
  • Sorted tool outputs & stable JSON serialization

9. Key Differentiators

Deterministic

Same input → identical output. Auditable AI for regulated domains.

Multi-LLM

OpenAI + Anthropic in production; Gemini at the edge. Hot-swappable per session.

Persona Crews

Three coordinated agent crews specialize Researcher / Analyst / Reporter behaviors.

Connector-First

GitHub, Jira, Confluence, Tavily, Arxiv built in.
